Phần 1: Lý thuyết (40 điểm)

Câu 1: Sự khác biệt giữa position: absolute và relative

- `position: relative`: Định vị phần tử tương đối so với vị trí bình thường của chính nó. Các phần tử khác xung quanh không bị ảnh hưởng (không bị lấp chỗ).
- `position: absolute`: Định vị phần tử tuyệt đối so với phần tử cha gần nhất có thuộc tính position khác `static`. Nó bị rút khỏi luồng (flow) bình thường của tài liệu, các phần tử khác sẽ tràn lên chiếm chỗ của nó.

- Ví dụ responsive: Đặt container ảnh là `relative`, icon "Sale" là `absolute` (top: 10px, right: 10px). Khi màn hình thu nhỏ, container co lại, icon vẫn luôn bám sát góc phải của ảnh thay vì bị trôi đi chỗ khác.

- Ảnh hưởng deploy: Các thuộc tính này xử lý ở phía Client (Browser). Khi deploy lên hosting, nó không ảnh hưởng tới server mà quyết định việc website có hiển thị đúng và không bị vỡ layout trên đa thiết bị (mobile, tablet, PC) hay không.

Câu 2: Quy trình đăng ký Domain và Cấu hình Hosting

- Quy trình: Mua tên miền (Domain) tại nhà cung cấp (GoDaddy, Namecheap) -> Đăng ký Hosting/Nền tảng deploy -> Trỏ NameServer (NS) hoặc bản ghi A/CNAME từ Domain về IP/Link của Hosting -> Đợi DNS cập nhật.

- Vai trò DNS: Hệ thống phân giải tên miền (DNS) giống như danh bạ điện thoại, dịch tên miền dễ nhớ (vd: abc.com) thành địa chỉ IP của server chứa mã nguồn, giúp trình duyệt tìm được website.

- 2 công cụ hỗ trợ: GitHub Pages, Netlify (hoặc Vercel).

- Rủi ro bảo mật: Nếu cấu hình DNS sai, website có thể bị trỏ nhầm sang trang web giả mạo (DNS Hijacking) để lừa đảo người dùng. Nếu không dùng HTTPS (SSL), dữ liệu người dùng có thể bị đánh cắp.

Câu 3: So sánh let, const, var trong JS

- Scope (Phạm vi): `var` có function scope (hoặc global). `let` và `const` có block scope (chỉ tồn tại trong cặp ngoặc nhọn `{}`).
- Hoisting: Cả 3 đều bị hoisted. Nhưng `var` được khởi tạo giá trị là `undefined`, còn `let` và `const` nằm trong "Temporal Dead Zone" (vùng chết tạm thời), nếu gọi trước khi khai báo sẽ gây lỗi ReferenceError.

- Ví dụ lỗi:
for(var i=0; i<3; i++) { setTimeout(() => console.log(i), 100); } -> In ra 3, 3, 3 vì var không giữ được scope trong vòng lặp bất đồng bộ. Sửa bằng cách thay `var` bằng `let`.

- Debug bằng Chrome DevTools: F12 mở DevTools -> Tab Sources -> Mở file JS -> Click vào số dòng để đặt Breakpoint. Tải lại trang, code sẽ dừng tại đó để ta kiểm tra giá trị của biến theo từng bước (Step over).

Câu 4: Event Delegation

- Khái niệm: Thay vì gắn sự kiện (Event Listener) cho từng phần tử con, ta gắn một sự kiện duy nhất cho phần tử cha chứa chúng. Khi con bị click, sự kiện sẽ "nổi bọt" (Event Bubbling) lên cha để xử lý.

- Ví dụ danh sách động:
document.getElementById('ul-list').addEventListener('click', e => { if(e.target.tagName === 'LI') console.log('Đã click', e.target.innerText); });

- Lợi ích tối ưu hiệu suất: Tiết kiệm RAM vì chỉ tạo 1 Listener thay vì hàng trăm Listener cho mỗi `li`. Đồng thời, khi DOM thay đổi (thêm phần tử HTML mới bằng JS), ta không cần phải gắn lại sự kiện cho các phần tử mới này.
